## 三大定律（低自由度 - 必须精确执行）

| 定律 | 规则 | 检查方式 |
|------|------|----------|
| **大纲即法律** | 严格执行大纲，不得擅自发挥 | 审查时对照大纲 |
| **设定即物理** | 实力/招式/物品 ≤ state.json | 写作前查询确认 |
| **发明需识别** | 新实体由 Data Agent 自动提取 | 章节完成后处理 |

## 新实体处理流程

v5.1 不再要求在正文中写 XML 标签。新实体由 Data Agent 在章节完成后自动提取：

1. **写作时**: 直接写纯正文，新角色/地点/物品正常描写
2. **完成后**: Data Agent 自动识别新实体并写入 index.db
3. **不确定实体**: Data Agent 标记为 uncertain，由人工确认

## 爽点密度要求

- 每章 ≥1 个小爽点（单一模式）
- 每 5 章 ≥1 个组合爽点（2种模式叠加）
- 每 10-15 章 ≥1 个里程碑爽点（改变主角地位）
- 避免连续 3 章同类型

## Strand 平衡警告

| 情节线 | 警告条件 |
|--------|----------|
| Quest（主线） | 连续 5+ 章 |
| Fire（感情线） | >10 章未出现 |
| Constellation（世界观） | >15 章未出现 |

## 禁止事项

- `[待补充]`、`[TODO]`、`...（省略）...` → 必须完整写出
- 战斗后无善后描述（都市异能题材）

**【剧情因果连贯 - 最高优先级】**
- 检查上一章结尾发生了什么，本章必须从那个时间点**紧密衔接**！
- **严禁跳过关键事件**！如果上一章是"战前准备"，本章就必须写"战斗过程"，不能直接跳到"战后总结"！
- 例如：上一章结尾敌人说"明日来战"，本章就必须写战斗，不能直接写"敌人已死"！
- 读者需要看到**过程**，而不是只看到**结果**！

**【角色状态一致性 - 极重要】**
- 仔细阅读上一章结尾中每个角色的状态（清醒/昏迷、位置、情绪等）！
- 本章开头必须延续上一章结尾的角色状态，**严禁矛盾**！
- ✗ 错误：上一章角色"虚弱地躺在床上听着声音"（已醒），下一章却写她"一直昏迷不醒"
- ✓ 正确：如果上一章角色已醒，下一章她应该仍然是醒着的

**【时间线逻辑约束 - 最高优先级】**

生理/物理事件必须遵循合理时间跨度，不能因大纲要求而硬凑：

| 事件类型 | 最小间隔要求 | 说明 |
|----------|------------|------|
| **怀孕→生产** | ≥40章（假设1章≈1天） | 十月怀胎，不可缩短 |
| **纳妾/同房→怀孕** | ≥5章 | 需要时间发现有孕 |
| **重伤→痊愈** | ≥3章（普通）/≥1章（金手指加速） | 需描写恢复过程 |
| **修炼→突破** | 按设定体系执行 | 不可无故加速 |

**若大纲要求与时间线冲突**：
- ❌ 错误做法：硬凑剧情（如15章就生孩子）
- ✅ 正确做法：在本章提示「时间线问题」，建议调整大纲节奏
- ✅ 替代方案：可写"已怀孕数月的角色"来追溯设定，但需在前文有暗示

**检查清单（写作前必查）**：
1. 若本章有"生产"剧情 → 检查该角色是否有"怀孕"记录，且间隔≥40章
2. 若本章有"怀孕"剧情 → 检查该角色"纳入/成婚"时间是否≥5章前
3. 若本章有角色"完全康复" → 检查"受伤"时间是否≥3章前

## 字数控制（红线 — 与大纲详略同等优先级）
- 目标：**3200-3800字**
- **严禁超过 4000 字**！(超过视为极严重失败)
- 严禁低于 3000 字
- **高密度策略**：面对包含 4-5 个以上情节事件的大纲，**必须使用紧凑笔法**。
- **详略得当**：全章只需 1-2 个高潮爽点做详细描写（单点<500字）。普通对话、移动、次要动作 1-2 句话快速带过，**绝不注水扩写**。
