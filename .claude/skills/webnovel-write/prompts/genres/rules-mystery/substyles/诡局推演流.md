你写的是**诡局推演流**——规则怪谈中最"烧脑"的类型。核心体验是：**主角像侦探一样在规则迷局中推理，读者跟着一起分析、猜测、恍然大悟。**

---

## 一、诡局推演流的灵魂：信息博弈

### 1.1 推理过程即内容

不是"告诉读者答案"，而是**展示完整的推理链条**：
- 输入：观察到的现象、获得的线索、收集的证词
- 处理：排除、关联、假设、验证
- 输出：阶段性判断（可能正确也可能需要修正）

### 1.2 每章的信息更新

每章必须给主角和读者**至少 2 条新信息**，推动认知前进：
- 一条确认或推翻之前的假说
- 一条打开新的推理方向

## 二、推理段落的写法

### 2.1 展示"怎么想到的"而不是"想到了什么"

- 差："他突然意识到，第三条规则其实是假的。"
- 好："第三条规则说'不准回头看'。但昨天张强回头了，什么也没发生。守则发下来的时间是下午三点——但张强回头的时间是上午十一点。如果规则的生效时间是从发下守则开始算……那之前的回头都不算违规。问题来了：还有多少规则，是有时间窗口的？"

### 2.2 逻辑链要可追溯

读者应该能跟着主角的思路走：
1. **前提 A**（观察到的事实）
2. **前提 B**（另一个事实/规则）
3. **推论 C**（A + B → 所以……）
4. **验证方式**（怎么证实/证伪这个推论）

### 2.3 红鲱鱼的使用

可以设置误导性线索，但**真线索必须在文中出现过**，回头能找到：
- 红鲱鱼占比不超过 30%
- 每条红鲱鱼在被排除时要给出合理解释
- 真正的线索要"藏在显眼的地方"——读者第一遍可能略过，回头一看"原来就在这里"

## 三、角色智力层级

### 3.1 主角：冷静、逻辑、有方法论

- 内心独白要有推理过程，不是直觉猜测
- 会犯错，但犯错后能快速修正
- 台词精准："你说昨天没去三楼。但你鞋底有三楼才有的红色粉末。"

### 3.2 配角：不同视角的信息源

- 有些配角提供关键线索但自己没意识到
- 有些配角有自己的推理（可能正确也可能错误）
- 配角之间的观点冲突推动推理深化

### 3.3 对话即推理战

诡局推演流的精彩对话是**观点交锋**：
```
"你觉得第五条是保护我们的？"
"至少表面上是。"
"那为什么遵守第五条的三个人里，有两个失踪了？"
"……你的意思是，遵守规则本身就是陷阱？"
"我不确定。但我确定一件事——守则的写作者不是为了让我们活下来。"
```

## 四、爽点设计

1. **推理正确爽**：主角的推断被完美验证，逻辑链闭合
2. **认知翻转爽**：旧推论被新证据颠覆，但新推论更合理更震撼
3. **信息差爽**：主角已经推理出了答案，但对手/同伴还不知道
4. **线索回收爽**：第一章的一句话在第五章突然变成了关键证据
5. **逻辑碾压爽**：主角用严密推理说服了所有人/堵死了对方的反驳

---

## 五、底线禁令

1. **禁止**作者直接公布答案——答案必须通过角色的推理和验证得出。
2. **禁止**无逻辑支撑的结论——每个判断都要有证据链。
3. **禁止**只堆氛围不推进认知——每章都要有新信息和新判断。
4. **禁止**纯靠直觉——主角可以有直觉，但必须后续用逻辑验证。
5. 章末必须让旧推论被撕开、或新线索彻底改写局面。
